TEoS beta 1.0 - Codebase Reference
==================================

Назначение файла
----------------
Единая текстовая документация по коду игры:
- какие модули есть в проекте,
- какие методы/функции за что отвечают,
- какие системы и механики используются,
- как данные проходят через игровой цикл.


1. Архитектура проекта
----------------------
Проект организован по подсистемам:
- `core/` - управляющая логика игры, цикл, конфиги, БД.
- `entities/` - игровые сущности (игрок, камень, рябь).
- `systems/` - генерация лабиринта, туман, звук, рендер, частицы.
- `ui/` - меню, HUD, экран победы.
- `utils/` - вспомогательные функции.
- `shaders/` - GLSL шейдеры воды/композита.
- `tests/` - базовые юнит-тесты.


2. Главный цикл и состояния
---------------------------
Файл: `core/game.py`
Класс: `Game`

Состояния:
- `STATE_MENU` - главное меню.
- `STATE_PLAYING` - игровой процесс.
- `STATE_PAUSED` - пауза.
- `STATE_SETTINGS` - настройки.
- `STATE_RECORDS` - отдельный экран рекордов.

Ключевые методы:
- `run()` - главный цикл приложения.
- `_collect_input()` - сбор событий мыши/клавиатуры.
- `_handle_keydown()` - горячие клавиши (ESC/F3/F5/F6).
- `_update_menu()`, `_draw_menu()`.
- `_update_settings()`, `_draw_settings()`.
- `_update_pause()`, `_draw_pause()`.
- `_update_records()`, `_draw_records()`.
- `_update_game()` - основная игровая логика.
- `_draw_game()` - основная игровая отрисовка.
- `_respawn()` - респавн после смерти.
- `_save_settings()` - сохранение пользовательских настроек.


3. Игровые механики
-------------------
3.1 Игрок
- `entities/player.py`, класс `Player`.
- Движение с нормализацией под FPS.
- Коллизии через набор контрольных точек.
- Смерть при касании стены (с учетом invincibility после респавна).

Ключевые методы:
- `update(keys, maze, dt, camera)` - движение и проверка смерти.
- `_collides(x, y, maze)` - проверка столкновения с стенами.
- `_distance_to_nearest_wall(x, y, maze)` - расстояние до ближайшей стены.
- `die()`, `respawn()`, `draw()`.

3.2 Камень
- `entities/stone.py`, класс `Stone`.
- Бросок с зарядкой, рикошет, затухающая траектория, подбор.
- Для уровня 3: пунктир и видимость следа через туман.

Ключевые методы:
- `start_charging()`, `update_charge(dt)`, `throw(...)`.
- `update(dt, maze)` - полет/столкновения/приземление.
- `_bounce(...)`, `_land(...)`, `can_pickup(...)`, `pickup()`.
- `draw(...)`.

3.3 Рябь
- `entities/ripple.py`.
- Рябь может быть скрыта до контакта с игроком.
- При "lit" ряби туман рассеивается кольцом с LOS.

Ключевые классы/методы:
- `Ripple`:
  - `update(dt, maze)`,
  - `check_player_contact(...)`,
  - `light_up()`,
  - `draw(...)`.
- `RippleManager`:
  - `create_ripples(...)`,
  - `update(...)`,
  - `get_visible_ripples()`,
  - `draw(...)`.


4. Уровни
---------
Конфиг: `settings.py`, `LEVEL_CONFIGS`.

Уровень 1:
- камень доступен,
- классическая ориентация по ряби.

Уровень 2:
- без камня,
- водные источники и светлячки.

Уровень 3:
- старт без камня,
- поиск "светящегося камня",
- ориентир на звук рикошетов и пунктир траектории,
- уменьшенный радиус видимости (`vision_mult`),
- мягкая подсветка пути при полете/лежании светящегося камня.


5. Генерация лабиринта
----------------------
Файл: `systems/maze.py`, класс `MazeGenerator`.

База:
- сеточный алгоритм с карвингом коридоров,
- обязательные `S` (старт) и `E` (выход).

Интеграция в `Game`:
- `_build_path_to_exit()` строит путь start->exit (BFS),
- `_create_glow_stone_spawn()` выбирает точку спавна камня на прогресс-участке пути.


6. Туман и видимость
--------------------
Файл: `systems/fog.py`, класс `FogOfWar`.

Логика:
- хранит `revealed`, `memory`, `active_cells`,
- `reveal_circle(...)` и `reveal_ring_with_los(...)`,
- `is_visible(...)` и `get_visibility(...)` с LOS.

Визуал:
- базовый тайловый туман (надежная логика),
- опционально "soft fog visual" в `Game._draw_soft_fog_visual(...)`:
  - плавные отверстия вокруг игрока/света/ряби,
  - уменьшает ощущение тайловости визуально.


7. Рендер и шейдеры
-------------------
Файл: `systems/renderer.py`, класс `Renderer`.

Подход:
- проход 1: OpenGL вода (`shaders/water.vert`, `shaders/water.frag`),
- проход 2: наложение Pygame-поверхности (`shaders/composite.frag`).

Ключевые методы:
- `update(dt)`,
- `set_camera(...)`,
- `set_ripples(...)`,
- `render(pygame_surface)`,
- `reload_shaders()`,
- `release()`.


8. Звук и частицы
-----------------
Звук:
- `systems/sound.py`, `SoundManager`.
- Пространственный звук (дистанция/панорама), шаги, рикошеты, splash, ambient.

Частицы:
- `systems/particles.py`, `ParticleSystem`.
- Смерть, победа, туман, визуальные акценты.


9. UI
-----
Файлы:
- `ui/components.py` - базовые элементы (`Button`, `ProgressBar`).
- `ui/menus.py` - `MainMenu`, `SettingsMenu`, `PauseMenu`, `RecordsMenu`.
- `ui/hud.py` - игровой HUD и `VictoryScreen`.

Особенности:
- отдельный экран рекордов,
- вывод лучшего рекорда на экране победы.


10. База данных
---------------
Файл: `core/run_database.py`, класс `RunDatabase`.
Движок: SQLite (`data/teos.db`).

Таблицы:
- `runs` - все успешные прохождения.
- `records` - лучший результат по (уровень, сложность).

Методы:
- `save_run(level, difficulty, timer)` -> bool (новый рекорд или нет),
- `get_best_record(level, difficulty)`,
- `get_top_runs(limit)`,
- `get_records_for_difficulty(difficulty)`,
- `get_top_runs_by_difficulty(difficulty, limit)`.


11. Утилиты
-----------
Файл: `utils/helpers.py`.

Функции:
- `raycast(...)` - луч до стен.
- `line_of_sight(...)` - проверка прямой видимости.
- `lerp_color(...)`, `lerp(...)`.


12. Точки расширения
--------------------
Рекомендуемые направления:
- экран расширенных рекордов (фильтры, сортировки, пагинация),
- хранение `seed` генерации в БД для повторяемых забегов,
- экспорт статистики в CSV,
- генератор лабиринта v2 (опорный путь + достройка ветвей),
- визуальный туман v2 с градиентными масками/шумом GPU.
